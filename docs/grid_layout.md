# Grid Layout & Playing Guide

## Overview

The grid interface in Seeker II is designed around three core principles:
1. Every note should be musically meaningful
2. Visual feedback should guide musical exploration
3. Consistent brightness levels convey system state

## Physical Layout

The grid is organized into distinct functional areas:

### Lane Controls (Four Corners)
Each corner contains a complete set of controls for one lane:
- **Stage Buttons**: Row of 4 buttons for stage selection/feedback
- **Transport Controls**: Play, Record, and Clear buttons
- Positioned for ergonomic access during performance

### Central Keyboard (6x6)
A musical keyboard occupies the center of the grid (coordinates 6-11, 2-7).

### Layout Overview
```
┌─────────────────────────┐
│ L1   Keyboard    L2    │
│ [S][S][S][S]    [S][S][S][S] │
│ [P][R][C]       [P][R][C]    │
│                         │
│     [Musical Keys]      │
│                         │
│ L3                L4    │
│ [S][S][S][S]    [S][S][S][S] │
│ [P][R][C]       [P][R][C]    │
└─────────────────────────┘
```

## Visual Feedback System

### Brightness Levels
- **BRIGHT (15)**: Maximum brightness, used for focused elements
- **ACTIVE (12)**: ~80% brightness, indicates active playback/recording
- **INACTIVE (9)**: ~60% brightness, for unfocused but available elements
- **MED (8)**: Default state for most buttons
- **DIM (2)**: Minimal visibility
- **OFF (0)**: Completely dark

### State Indication
1. **Focused Lane**
   - All controls at BRIGHT level
   - Stage buttons show current position
   - Transport controls clearly visible

2. **Active States**
   - Playing: Button pulses at ACTIVE brightness
   - Recording: Button pulses between MED and ACTIVE at tempo
   - Current stage: BRIGHT when focused, MED when unfocused
   - Active notes: Stage pulses ACTIVE when notes are playing

3. **Unfocused Lanes**
   - Controls at MED level
   - Maintain visibility while clearly indicating inactive state

4. **Background Animation**
   - Subtle star-field effect
   - Provides ambient movement without distracting
   - Helps identify active grid connection

## Lane Controls

### Stage Buttons
- Four buttons showing available stages
- Current stage highlighted
- Pulses when notes are playing
- BRIGHT when focused, MED when unfocused

### Transport Controls
1. **Play Button**
   - ACTIVE when playing
   - BRIGHT when focused
   - MED when inactive

2. **Record Button**
   - Pulses at tempo when recording
   - BRIGHT when focused
   - MED when inactive

3. **Clear Button**
   - BRIGHT when focused
   - MED when unfocused
   - Immediate response for safety

## Performance Techniques

### Focus Management
- Each lane maintains independent focus state
- Clear visual feedback about current focus
- Quick switching between lanes during performance

### Visual Monitoring
- Active playback clearly visible across all lanes
- Recording state immediately apparent
- Stage progression easily tracked
- Note activity reflected in stage brightness

### Ergonomic Design
- Controls positioned for easy access
- Consistent layout in each corner
- Clear separation between lanes
- Central keyboard area remains uncluttered

## Musical Keyboard (Center)

### Design Philosophy
The keyboard uses a Modal Tonnetz layout where:
- Every position is in key
- Geometric shapes create consistent musical phrases
- Visual patterns match musical patterns

### Movement Guide
- **→ Right**: Move up by thirds
- **↓ Down**: Move up by seconds
- **↗ Diagonal**: Wider intervallic movement
- **Root Notes**: Form a repeating geometric pattern

### Three Voice System
1. **Top Row**: Harmony voice (thirds/sixths above)
2. **Middle Row**: Main melody and chord roots
3. **Bottom Row**: Counterpoint (contrary motion)

## Visual Feedback

### Lane State
- **Active Lane**: Brighter controls and stage display
- **Recording**: Pulsing rec button
- **Playing**: Animated play button
- **Stage Progress**: Moving lights across stage display

### Global Elements
- **Background**: Subtle glittering animation
- **Static Areas**: Defined spaces for keyboard and controls
- **Lane Boundaries**: Clear visual separation between lanes

## Playing Techniques

### Basic Patterns
1. **Melodic Lines**
   - Follow any path for musical phrases
   - Use diagonal movement for wider intervals
   - Combine horizontal/vertical for varied melodies

2. **Harmonic Patterns**
   - Stack notes vertically for chords
   - Use diagonal shapes for extended harmonies
   - Follow root patterns for chord progressions

### Advanced Techniques
1. **Multi-Voice Patterns**
   - Hold middle + top row for instant harmonies
   - Use bottom row for counterpoint
   - Create polyrhythms across voices

2. **Geometric Shapes**
   - Triangles: Triadic patterns
   - Rectangles: Fourth/fifth patterns
   - Diamonds: Extended chord voicings

## Practice Suggestions

1. **Start Simple**
   - Learn the root note pattern
   - Practice basic horizontal/vertical movements
   - Experiment with simple shapes

2. **Build Complexity**
   - Add second voice harmonies
   - Explore counterpoint patterns
   - Combine multiple techniques

3. **Performance Skills**
   - Practice transitioning between patterns
   - Experiment with transform timing
   - Develop muscle memory for common shapes
